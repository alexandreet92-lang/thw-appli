# Inventaire des features — présentation utilisateurs

## Objectif
Créer `FEATURES_USER.md` à la racine : liste de TOUTES les fonctionnalités de
l'app THW Coaching, regroupées par THÈMES, pour les futurs utilisateurs du site.
Pas un audit technique — « voici tout ce que l'app sait faire, simplement ».

## Public & ton
Futur utilisateur qui veut tout savoir avant de s'inscrire. Ton direct, positif,
factuel, concret, vendeur sans être commercial, sans jargon.

## Structure
Par grands thèmes : titre clair + intro courte (1-2 phrases) + liste des features
(nom + description 1-3 phrases). Thèmes : Coach IA, Compétences, Actions rapides,
Analyse de performances, Planification, Calendrier, Récupération, Nutrition,
Connexions, Mon profil, Tokens, Abonnements, Notifications, Personnalisation,
Enregistrement/Séances, Sécurité.

## Règles
NE PAS : lister bugs/zones d'ombre, commenter le partiel, faire de l'analyse
technique, citer des noms de tables/fichiers. JUSTE : présenter les features comme
un produit fini, expliquées en quelques mots, avec exemples concrets.

## Format
Markdown propre (## thèmes, ### sous-thèmes), emojis légers, listes à puces,
texte court. Longueur libre, mais chaque feature ≤ 3 phrases.

## Livrable
`FEATURES_USER.md` + commit « docs: user-facing features inventory ».
